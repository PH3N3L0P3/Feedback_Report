05 — Risk / Things to Watch
ความเสี่ยงถ้าไม่แก้ไข
Critical
Pay-to-win perception ลุกลาม
F2P + Casual เชื่อว่าเกมไม่แฟร์ → churn สูง → ฐานผู้เล่นหดตัว → รายได้ระยะยาวลดลงแม้ Whale ยังอยู่
Critical
Bug สะสมทำ session drop
Lag + เสียงหาย + display เพี้ยน → ผู้เล่นหยุดเล่นกลางเกม → DAU และ session length ลดลงอย่างต่อเนื่อง
Watch
Gacha trust eroding
Pity ไม่โปร่งใส + banner ถี่ → Mid Spender + Whale รู้สึก "ไม่คุ้ม" → ยอด gacha revenue ลดลงระยะยาว
Watch
New Player retention ต่ำ
Tutorial เร็ว + ด่านยาก + reward น้อย → New Player drop ใน first week สูง → user base ไม่เติบโต
Monitor
Event fatigue
ภารกิจซ้ำ + grind เยอะ → participation ลดลง → ลงทุน event แต่ได้ engagement คืนน้อย

07 — Workflow Note
AI Workflow ที่ใช้วิเคราะห์
STEP 01
Data Preparation
Extract 300 rows จาก Excel ด้วย Python — ตรวจ Missing Values, แปลง date serial → YYYY-MM-DD, จัด columns ตาม Data Dictionary
STEP 02
AI Classification
วิเคราะห์ทุก 300 feedback — สร้าง sentiment, category, priority, ai_summary, suggested_owner โดย rule-based + AI inference
STEP 03
Sentiment Rules
รบกวน = Neutral · อยากให้/ควรมี/ถ้าแก้ได้ = Positive · ขอบคุณ = Positive · อารมณ์ชัดเจน = Negative
STEP 04
Priority Rules
High = Bug + Account/Payment + เพราะกระทบการเล่น · Low = Event + Positive Feedback · Medium = ที่เหลือ
STEP 05
game_area Inference
AI เติม 99 rows ที่ว่าง โดย keyword matching จากเนื้อหา feedback — แสดงสีม่วงแยกจาก Dataset เดิม
STEP 06
Insight Generation
วิเคราะห์ pattern — สรุป Top Issues, Root Cause, Impact, Recommended Actions แยกตามทีม
STEP 07
Human Review
ตรวจสอบ AI output และเพิ่ม review_note ในคอลัมน์ Human Review — สุ่มตรวจ >10% (30+ รายการ)
STEP 08
Report Output
สร้าง Short Report (.docx + .html) และ Insight Summary (.xlsx) ครบ 8 sections ตาม template ที่กำหนด
